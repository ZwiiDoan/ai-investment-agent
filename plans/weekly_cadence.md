# Weekly Study Cadence (1h/day)

## Breakdown by Day

- **Day 1 — System Design (1h):**
  Read design topic, diagram component, update ADR.

- **Day 2 — Core Backend Coding (1h):**
  Python/FastAPI or Go service feature.

- **Day 3 — AI Integration (1h):**
  LLM calls, vector DB queries, orchestration experiments.

- **Day 4 — Advanced Coding (1h):**
  Rust module or observability improvements.

- **Day 5 — Documentation & ADRs (1h):**
  Write/refine ADR (1–2 pages).

- **Day 6 — Frontend Literacy (1h):**
  Next.js basics: auth, API calls, pagination.

- **Day 7 — Reflection & Planning (1h):**
  Review progress, adjust tasks, journal entry.

## Ratio

- 3h coding (Python/Go/Rust)
- 2h system design & documentation
- 1h AI exploration
- 1h frontend demo literacy

## Weekly Output

- One demo, ADR, or diagram every week.
